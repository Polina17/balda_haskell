# balda. Правила игры
#
# Начало игры:
# 1. Выбирается исходное слово из заданного словаря, определяется его длина n.
# 2. Строится размеченное поле размером n*n клеток. 
#    Прим.: Слово определяется такой длины, чтобы кол-во пустых клеток (n*n - n) делилось нацело на кол-во игроков - тогда  у каждого игрока будет одинаковое кол-во ходов и одинаковая возможность набрать баллы (в случае отсутствия пропуска ходов). 
# 3. Буквы исходного слова вписываются в клетки строки n/2 (при чётном n) или (n+1)/2 (при нечётном n), считая сверху. Таким образом, исходное слово займёт все клетки серединной строки.
# 4. Кол-во игроков - 2-4. 
# 5. У каждого игрока изначально - 0 баллов. Кол-во пропусков ходов у каждого - 0.
#
# Ход игры:
# 6. Варианты хода для каждого игрока:
#    a. Добавить 1 букву в клетку, граничащую по вертикали/горизонтали с хотя бы одной уже заполненной клеткой так, чтобы образовалась неразрывная и несамопересекающаяся ломаная линия (ломаться может только по вертикали/горизонтали), представляющая собой слово из Толкового словаря. Добавленная буква - не обязательно начало или окончание образовавшегося слова. Слово только в нарицательное существительное, в Им.П., ед.ч. (допустимый словарь будет задан в программе);
#    b. Пропустить ход.
#
# 7. Что меняется после хода:
#   В случае 6a:
#    a. Добавляется 1 буква в клетку, граничащую по вертикали/горизонтали с хотя бы одной уже заполненной клеткой (кол-во пустых клеток уменьшается на 1);
#    b. Данному игроку добавляется кол-во баллов, равное длине ломаной линии (длине слова);
#    c. Ход передаётся другому игроку.
#   В случае 6b:
#    a. Поле не меняется;
#    b. Кол-во пропусков ходов увеличивается на 1 у игрока, чей был ход;
#    c. Кол-во баллов уменьшается на 1 у игрока, чей был ход (штрафной балл за пропуск хода).
#
# Окончание игры:
# 8. Если не было пропусков хода, то игра может закончиться при одном из условий:
#     a. Пустых клеток нет (у всех игроков было одинаковое кол-во ходов);
#     b. Пустые клетки есть, но слово составить невозможно при получившемся раскладе букв.
#    Если были пропуски хода, то игра может закончиться при одном из условий:
#     a. Если на момент очередного хода 1-го игрока пустых клеток меньше, чем игроков;
#     b. Пустые клетки есть, но слово составить невозможно при получившемся раскладе букв.
#     c. Если у каждого игрока набирается 3 пропуска игры (ничья)
# 9. Считается сумма баллов у каждого игрока. Выиграл тот, у кого больше баллов, за исключением случая, когда фиксируется ничья (3 пропуска хода у каждого игрока).
#
#
#
# Типы: 
# 1. Поле - набор клеток;
# 2. Слово - набор букв (алфавит задан);
# 3. Игрок (по умолчанию 2 игрока) - характеризуется:
#    a. Кол-вом баллов (изначально 0);
#    b. Кол-вом пропусков хода (изначально 0);
#    c. Номером порядка хождения (кто ходит 1-й, кто 2-й);
# 4. Ход:
#    a. Добавление буквы согласно правилам (см. выше). Словарь допустимых слов задан (Толковый или выборка из него);
#    b. Пропуск хода;
# 5. Условие завершения - параметры: кол-вом пустых клеток, возможность составить слово
#    1. Кол-во пустых клеток = 0 при отсутствии пропусков (true/false);
#    2. Возможность составить слово ??
#    3. Кол-во пропусков хода = 3 у обоих игроков (true/false) и т.п.
#
#
# Функции:
# 1. Смена поля (вызывается вследствие хода, выводит поле с добавлением буквы или отсутствием изменений)
# 2. Смена характеристик игрока (вызывается вследствие хода, выводит изменённый балл и изменённое кол-во пропусков хода)
# 3. Смена значений параметров завершения (вызывается вследствие хода, выводит изменённые параметры завершения)
# 4. Завершение (вызывается при значении true хотя бы 1-го из параметров завершения, выводит кол-во баллов у каждого игрока, результат - номер поведителя или ничья)  
#
